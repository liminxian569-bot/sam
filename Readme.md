## 🏗️ 核心架構與分工

### 1. WordBank (題目與資料管理)
### 負責人：吳宸緯

* **角色定位：** 負責處理遊戲的核心資料與題目生成邏輯。
* **技術要點：**
    * 管理靜態單字題庫 (`std::vector<string>`)。
    * 負責隨機抽題邏輯，確保每局遊戲的隨機性。
    * **挖空演算法：** 根據單字長度與結構，自動執行動態挖空邏輯。
    * **封裝設計：** 隱藏原始單字庫，僅透過介面提供題目與答案。

### 2. GameManager (遊戲邏輯與 UI 呈現)
### 負責人：李旻憲

* **角色定位：** 擔任遊戲「裁判」，同時負責所有畫面輸出與顯示。
* **技術要點：**
    * **狀態控管：** 初始化玩家血量，並在猜錯時執行扣血判定。
    * **比對機制：** 即時比對玩家輸入與正確答案，更新題目顯示狀態。
    * **錯誤紀錄：** 儲存已猜過的錯誤字母，避免重複輸入導致扣血。
    * **視覺輸出：** 負責繪製 ASCII 介面、顯示目前題目狀態與剩餘血量。
    * **勝負判定：** 監控遊戲是否達到勝利（單字補全）或失敗（血量歸零）條件。

### 3. Main (程式主流程與防呆機制)
### 負責人：李庭偉

* **角色定位：** 串聯各模組，並負責處理使用者的輸入例外。
* **技術要點：**
    * **流程控制：** 初始化 `WordBank` 與 `GameManager` 物件，引導遊戲主迴圈。
    * **強大防呆 (Input Validation)：**
        * **長度驗證：** 強制限制玩家僅能輸入單一字元，避免系統崩潰。
        * **類型驗證：** 使用 `isalpha` 確保輸入為英文字母，排除數字或符號。
        * **重複檢查：** 自動判斷該字母是否已被猜過。
    * **格式處理：** 使用 `tolower` 統一處理大小寫轉換。

## 🕹️ 遊戲規則說明

### 1. 遊戲開始
系統會從題庫隨機抽取單字，並以隱藏模式顯示（例如：`c_mp_ter`），底線代表玩家需要猜測的字母。

### 2. 玩家操作
玩家每次輸入 **一個英文字母**。為了保持畫面整潔，系統在每次輸入後會自動清空螢幕 (`system("cls")`) 以顯示最新狀態。

### 3. 判定邏輯
* **猜對了：** 對應位置的底線轉為正確字母，玩家血量不變。
* **猜錯了：** 扣除 1 點血量，且該字母會記錄在「錯誤區」提醒玩家。
* **無效輸入：** 若輸入多個字元、數字或重複字母，系統會給予警告且不扣血。

### 4. 勝負判定
* **恭喜獲勝：** 在血量歸零前補全單字。
* **遊戲失敗：** 猜錯 3 次（血量歸零）即結束，系統將公佈正確答案。
