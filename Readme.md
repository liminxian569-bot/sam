# 🎮 物件導向猜字遊戲專案報告 
## 組長：李旻憲
## 組員：李庭偉、吳宸緯

這份專案是基於 C++ 物件導向程式設計（OOP）開發的猜字遊戲。我們將系統拆解為資料、邏輯與控制三大核心模組，並透過多檔案編譯達成高效的開發流程。

---

## 🏗️ 模組職責細節

### 1. WordBank (資料與題目生成)
* **角色：** 遊戲的「資料庫」與「題目製作人」。
* **負責人:吳宸緯
* **詳細職責：**
    * 管理靜態單字題庫 (`std::vector<string>`)。
    * 設定隨機種子 (`srand`)，確保每局遊戲的隨機性。
    * **核心演算法：** 根據單字長度與奇偶性，自動執行不同的挖空邏輯。
    * **封裝設計：** 隱藏原始單字庫，僅透過介面提供答案與題目。

### 2. GameManager (遊戲邏輯與顯示)
* **角色：** 遊戲的「裁判」與「顯示器」。
* **負責人:李旻憲
* **詳細職責：**
    * **生命值管理：** 初始化血量並在玩家猜錯時執行扣血。
    * **比對機制：** 逐一比對玩家輸入的字母與正確答案，並更新題目顯示狀態。
    * **歷史紀錄：** 儲存玩家所有「猜錯」的字母，避免玩家重複輸入。
    * **UI 輸出：** 負責繪製 ASCII 邊框、顯示目前的題目狀態、剩餘血量（`<3`）與錯誤紀錄。
    * **結算判定：** 監控遊戲是否達到勝利（全字拼完）或失敗（血量歸零）條件。

### 3. Main 函式 (主程式控制與防呆)
* **角色：** 整個遊戲的「大腦」與「守門員」。
* **負責人:李庭偉
* **詳細職責：**
    * **流程控制：** 初始化 `WordBank` 與 `GameManager` 物件，並維持遊戲主迴圈。
    * **強大防呆 (Validation)：**
        * **長度驗證：** 限制玩家只能輸入單一字元，避免程式崩潰。
        * **類型驗證：** 使用 `isalpha` 確保輸入是英文字母而非數字或符號。
        * **重複驗證：** 檢查該字母是否已被猜過，避免無謂的血量扣除。
    * **格式處理：** 使用 `tolower` 將所有輸入統一轉為小寫。

## 🕹️ 遊戲玩法說明 (Game Rules)

### 1. 遊戲開始
* 系統隨機抽取一個單字。
* 畫面會顯示該單字的「隱藏版本」（例如 `c_mp_ter`），底線代表需要猜測的字母。

### 2. 玩家操作
* 玩家每次輸入 **一個英文字母** 並按 Enter。
* 輸入後，系統會自動清空畫面 (`system("cls")`) 以顯示最新狀態。

### 3. 判定機制
* **猜對了：** 題目中對應的底線會顯示為正確字母，血量不變。
* **猜錯了：** 扣除 1 滴血，且該字母會出現在「Wrong Guesses」區提醒玩家。
* **無效輸入：** 輸入數字、多個字元或重複字母，系統會顯示警告而不扣血。

### 4. 勝負判定
* **恭喜獲勝：** 在血量歸零前補全單字。
* **遊戲失敗：** 猜錯 3 次（血量歸零）。系統會公佈正確答案。
